Title: take_quiz

Primary Actor: User

Stakeholders and Interests:

User : Access a quiz, navigate questions of the quiz.

Precondition: User has to be logged in the system.

Trigger: User tries to access to the quiz.

Main Scenario:

  1. User tries to access to the quiz.
  2. SuD checks status of the user, time, and attmpts.
  3. SuD completes the accessbility check.
  4. User is permitted to proceed the quiz.
  5. User navigates the questiosn.
  6. SuD shows the questions that the user requested.
  7. User submits the quiz.

Extensions:

2a. There does not exists criteria set up .

2a1. Default criteria are applied.

2a2. SuD inquiries users new criteria for the access limitations.

Variation:

5a. User can saves the answers.

5b. User can suspends current attempt for later submission.

7a. All the questions are not answered.

7a1. Warn the user to complete all questions before the submission.
